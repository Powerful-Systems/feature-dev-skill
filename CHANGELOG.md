# Changelog

All notable changes to the Feature Development Workflow skill will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - 2026-01-28

### Added
- Initial release of Feature Development Workflow skill
- 10-phase structured workflow from issue to PR
- Support for both Linear and GitHub Issues
- Competing hypotheses pattern in Phase 1
- Self-validation checks in every phase
- Parallel execution support in Phase 4
- Integration with code-simplifier, frontend-design, web-design-guidelines skills
- Comprehensive documentation (README, EXAMPLES, CONTRIBUTING)
- Customization guide for project-specific adaptations
- Stopping conditions and fail-fast mechanisms
- State tracking for long-horizon sessions
- GitHub issue command reference

### Core Phases
1. Requirements Discovery
2. Implementation Planning
3. Simplification Review
4. Implementation
5. Initial Testing
6. Code Simplification
7. Design Review
8. Comprehensive Testing
9. Pull Request
10. Cleanup & Documentation

### Documentation
- Complete README with installation and usage
- EXAMPLES.md with 5 real-world scenarios
- CONTRIBUTING.md with contribution guidelines
- Detailed customization instructions
- Quick reference for GitHub CLI commands
